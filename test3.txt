1) Процесс реагирования (NIST SP 800-61, упрощённо)
 - Preparation (Подготовка): политики, доступы, инструменты, образы для анализа, список контактов.
 - Detection and Analysis (Обнаружение и анализ): собрать лог-файлы, pcap, сигнатуры, идентифицировать тип атаки.
 - Containment, Eradication and Recovery (Локализация, устранение, восстановление): временные и окончательные меры, восстановление из бэкапов.
 - Post-Incident Activity (После инцидента): отчёт, уроки, улучшение процедур.

В этом модуле B ожидается: быстрый и точный форензик, корректная временная линия, оценка масштаба и грамотные рекомендации на основе NIST.

4) Какие логи за что отвечают — карта (очень полезно для Module B)
(обозначения: путь на Linux/Unix и эквивалент/описание для Windows)

Системные логи:
 - /var/log/auth.log или /var/log/secure — аутентификация/ssh/логины (Linux). Windows: Security Event Log (ID 4624, 4625 — logon/logoff).
 - /var/log/syslog или /var/log/messages — системные события/демоны/сервисные сообщения. Windows: System Event Log (System).
 - journalctl (systemd) — агрегат systemd-событий.

Веб-серверы:
 - /var/log/nginx/access.log — HTTP доступы (IP, URL, код, user-agent, referrer).
 - /var/log/nginx/error.log — ошибки обработки запросов, внутренние ошибки сервера.
 - /var/log/apache2/access.log — аналогично для Apache.
 - IIS: %SystemDrive%\inetpub\logs\LogFiles\W3SVC... — лог доступа IIS.

Приложения / API:
 - /var/log/<app>/app.log — application-specific лог (часто JSON). Часто содержит ошибки, stack traces, user actions.
 - Windows: приложения пишут в Application Event Log.

Аудит и OS-level (SELinux, auditd):
 - /var/log/audit/audit.log — SELinux/auditd events (sudo usage, file access audit). Windows: Audit logs + Advanced Audit Policy.

Сетевой уровень:
 - PCAP файлы (capture_full.pcap) — сырые пакеты. Используются для глубокого анализа с Wireshark.
 - Zeek logs (conn.log, http.log, dns.log) — удобны для быстрого обзора сессий и HTTP/DNS активностей.
 - Suricata/IDS alerts — сигнатуры сработавшие по трафику (alerts.log, eve.json).

Аутентификация и идентификация пользователей:
 - /var/log/secure, /var/log/auth.log — ssh auth, sudo. Windows: Security log, Event IDs (4624 — вход, 4625 — неудачный вход, 4672 — special privileges).

Процедура поиска релевантных логов:
 - Определить время инцидента (T0) — использовать первые индикаторы (alert, user report).
 - Собирать логи +/- 48 часов вокруг T0 (или больше) для построения контекста.
 - Конвертировать временные зоны в UTC и нормализовать метки времени.

5) Анализ сетевого трафика (Wireshark / tshark / Zeek / Suricata)
Инструменты: Wireshark (GUI), tshark (CLI), Zeek (Bro), Suricata (IDS), capshark-like utilities.

5.1. Первичный обзор pcap
# Обзор в tshark: top talkers (IP), портов, протоколов
tshark -r evidence/capture_full.pcap -q -z io,stat,0,ip
tshark -r evidence/capture_full.pcap -q -z conv,ip
# Вывести HTTP запросы в readable виде
tshark -r evidence/capture_full.pcap -Y http.request -T fields -e ip.src -e ip.dst -e http.request.method -e http.request.uri -e http.user_agent
# Извлечь файлы из pcap (HTTP objects)
tshark -r evidence/capture_full.pcap --export-objects "http,./evidence/http_objs"
# Фильтры Wireshark (display filters)
# Примеры: ip.addr == 198.51.100.10 && tcp.port == 80
# DNS suspicious: dns.qry.name contains "exfil"
# HTTP status codes: http.response.code == 500

5.2. Zeek (Bro) — быстрая обработка pcap в лог форматы
# Запуск Zeek на pcap:
zeek -r evidence/capture_full.pcap
# Результат: conn.log, http.log, dns.log, files.log, ssl.log и др.
# Анализ conn.log:
cat conn.log | bro-cut id.orig_h id.resp_h proto service duration orig_bytes resp_bytes
# Поиск сессий к подозрительным хостам
grep "198.51.100.10" conn.log

5.3. Suricata/Eve JSON
# Запуск Suricata на pcap
suricata -r evidence/capture_full.pcap -l evidence/suricata_output
# eve.json содержит события, HTTP/DNS/alerts — удобно для jq фильтрации
jq '. | select(.alert)' evidence/suricata_output/eve.json

5.4. YARA для вредоносных payloads
# Запуск yara по извлечённым файлам
yara -r rules.yar evidence/http_objs/

9) Оценка ущерба и план восстановления
 - Оценка масштаба: количество затронутых хостов, учетных записей, объема утекших данных (PII, финансовая информация).

   * Краткосрочная: изоляция хоста, блокировка IP/URL на firewall, смена учетных паролей/ключей, останов сервиса.

 - Eradication (искоренение): удалить бэкдоры, вредоносные файлы, откатить изменения (из известных clean backups).
 - Recovery: восстановление из бэкапов, тестирование integrity, упрощённый очередной мониторинг.
 - Документировать все шаги, создавать чек-листы восстановления и верификации.

# 1) extract_iocs.py — извлекает IP, URLs и хэши из текстовых логов
import re, sys, os, json
def extract_iocs(log_dir='evidence', out_dir='evidence/iocs'):
    if not os.path.exists(out_dir):
        os.makedirs(out_dir)
    ip_re = re.compile(r'([0-9]{1,3}\.){3}[0-9]{1,3}')
    url_re = re.compile(r'(http|https)://[^\s\'"]+')
    sha256_re = re.compile(r'\b[a-fA-F0-9]{64}\b')
    ips=set(); urls=set(); hashes=set()
    for fname in os.listdir(log_dir):
        path=os.path.join(log_dir,fname)
        if os.path.isfile(path):
            try:
                with open(path,'r',errors='ignore') as f:
                    for line in f:
                        for ip in ip_re.findall(line):
                            ips.add('.'.join(ip[0:4]) if isinstance(ip, tuple) else ip)
                        for url in url_re.findall(line):
                            urls.add(url)
                        for h in sha256_re.findall(line):
                            hashes.add(h)
            except Exception as e:
                pass
    with open(os.path.join(out_dir,'ips.txt'),'w') as f: f.write('\\n'.join(sorted(ips)))
    with open(os.path.join(out_dir,'urls.txt'),'w') as f: f.write('\\n'.join(sorted(urls)))
    with open(os.path.join(out_dir,'hashes.txt'),'w') as f: f.write('\\n'.join(sorted(hashes)))
if __name__=='__main__':
    extract_iocs()

# 2) merge_csv_timeline.py — объединяет CSV-файлы по полю timestamp и сортирует
import csv, glob, os, datetime, sys
def parse_iso(ts):
    # Попытка парсинга разных форматов, возвращает объект datetime
    for fmt in ('%Y-%m-%dT%H:%M:%S','%Y-%m-%d %H:%M:%S','%d/%b/%Y:%H:%M:%S'):
        try:
            return datetime.datetime.strptime(ts,fmt)
        except:
            pass
    # fallback: try epoch
    try:
        return datetime.datetime.utcfromtimestamp(float(ts))
    except:
        return None
def merge_csv(dir_in='evidence/csv_logs', out='evidence/timeline_merged.csv'):
    rows=[]
    for f in glob.glob(os.path.join(dir_in,'*.csv')):
        with open(f,'r',errors='ignore') as fh:
            reader=csv.DictReader(fh)
            for r in reader:
                ts=parse_iso(r.get('timestamp',r.get('time',r.get('date',''))))
                if ts:
                    rows.append((ts.isoformat(), r))
    rows.sort(key=lambda x: x[0])
    # write merged CSV with original data in 'payload' column
    with open(out,'w',newline='') as outfh:
        fieldnames=['timestamp','source','event_type','payload']
        writer=csv.DictWriter(outfh,fieldnames=fieldnames)
        writer.writeheader()
        for ts,r in rows:
            writer.writerow({'timestamp':ts,'source':r.get('source','unknown'),'event_type':r.get('event_type','log'),'payload':json.dumps(r)})
if __name__=='__main__':
    merge_csv()

# 3) pcap_summary.py — использует tshark для сбора summary (нужен tshark в PATH)
import subprocess, json, sys, os
def pcap_summary(pcap='evidence/capture_full.pcap', out='evidence/pcap_summary.json'):
    cmd=['tshark','-r',pcap,'-q','-z','io,stat,0,COUNT']
    try:
        p=subprocess.check_output(cmd,stderr=subprocess.STDOUT).decode(errors='ignore')
    except Exception as e:
        p=str(e)
    with open(out,'w',encoding='utf-8') as f:
        f.write(p)
if __name__=='__main__':
    pcap_summary()
